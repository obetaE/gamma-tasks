Solving Image Quality Issues in Web Design

Learn how to find and use high-quality images that look great on all screen sizes

The Image Quality Problem

When working with responsive designs using vh/vw units, images often appear blurry or pixelated because:

1Small images are being stretched

Using a 600px wide image in a container that expands to 1500px on large screens will always result in poor quality.

2Wrong image format selection

Using JPEG for graphics with sharp edges or PNG for photographs can result in larger file sizes or artifacts.

3Incorrect cropping

Images not optimized for different aspect ratios can appear distorted on various screen sizes.

Search Tips

When searching for images, use terms like "high resolution", "4K", "UHD", or "large" to find better quality images. Always check the actual dimensions before downloading.

Understanding Image Dimensions

Small Image (400px)

Good for mobile thumbnails but will pixelate when expanded.

Medium Image (800px)

Works for most content areas but may struggle on large screens.

Large Image (1600px)

Ideal for full-width hero sections on large screens.

Rule of Thumb for Image Sizes

For containers using vw/vh units, your image should be at least 1.5x the maximum expected display size. For a container that might expand to 1200px, use an image that's at least 1800px wide.

Implementation Solutions for Next.js

Next.js Image Component

Next.js provides an optimized Image component that handles responsive images automatically:

<Image
  src="/flower-image.jpg"
  alt="Beautiful flowers"
  width={1200}
  height={800}
  layout="responsive"
/>

Using multiple image sizes with srcSet

For background images, you can use the picture element to provide different sources:

<picture>
  <source media="(min-width: 1200px)" srcSet="/large-flower-image.jpg">
  <source media="(min-width: 768px)" srcSet="/medium-flower-image.jpg">
  <img src="/small-flower-image.jpg" alt="Flowers">
</picture>

CSS Solutions for Background Images

When using background images with vh/vw units:

.hero-section {
  background-image: url('/flower-bg-large.jpg');
  background-size: cover;
  background-position: center;
  height: 100vh;
  width: 100vw;
}

/* For smaller screens */
@media (max-width: 768px) {
  .hero-section {
    background-image: url('/flower-bg-small.jpg');
  }
}

Practical Tips for FloralMeets

1. Create image size standards

Define standard image sizes for different use cases:

  • Hero images: 2000px width
  • Product images: 1200px width
  • Thumbnails: 400px width

2. Optimize images before uploading

Use tools like Squoosh, ImageOptim, or Photoshop to compress images without noticeable quality loss before adding them to your Next.js project.

3. Use CSS object-fit for consistent cropping

When working with images in variable aspect ratios:

.product-image {
  width: 100%;
  height: 300px;
  object-fit: cover;
}

4. Implement lazy loading

Use Next.js's built-in lazy loading or the loading="lazy" attribute to improve performance:

<Image
  src="/flower.jpg"
  alt="Flowers"
  width={800}
  height={600}
  loading="lazy"
/>
